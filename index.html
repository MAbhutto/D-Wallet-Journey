Design Diagram

1. Data Sources

  • Transactions: Transactional data of all kinds of services
  • Users: User information
  • MSU customer metrics: Data is queried to view the monthly status of users with their transactions across all services
  • MSU combo details: Data is queried to view the number of users across all combinations of services every month
  • MSU graduation details: Data is queried to see how many days it took for each user to graduate as a multi-service user

Description: These data sources provide raw transactional data necessary for the analysis. Each service generates data that reflects user interactions and transactions.

2. ETL Process

  • Extract: Collecting data from various sources.
  • Transform: Cleaning, normalizing, and enriching data to ensure consistency and quality.
  • Load: Loading the transformed data into the data warehouse.

Description: The ETL process ensures that raw data from different sources is accurately collected, processed, and stored in a unified format, making it ready for analysis.

3. Data Storage Infrastructure

Unified Data Repository: A centralized storage system where all processed data is stored on SQL server.

Description: This component serves as the main data storage hub, consolidating data from multiple sources into a single repository for easy access and analysis.

4. Data Analysis

  • Descriptive Analysis: Analyzing historical data to understand past behaviors and trends.
  • Segmentation: Identifying distinct user personas and segmenting users based on their behaviors.

Description: This module performs various types of analysis to derive insights from the data, including understanding historical trends, predicting future outcomes, and segmenting users for targeted strategies.

5. Interactive BI Tools

Power BI Dashboards for Stakeholders: Visual dashboards created using Power BI to present data insights in an interactive and understandable format.

Description: Dashboards provide a visual representation of the analysis, allowing stakeholders to interact with the data, apply filters, and drill down into specific metrics. This facilitates better understanding and decision-making.

By following this design methodology, the project ensures a structured approach to data analysis, enabling the generation of valuable insights that can drive strategic decisions and optimize the services offered by the wallet business.
